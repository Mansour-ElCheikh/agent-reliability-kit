# epic.md

**Status:** Approved

## Problem
Re-running the analyzer recomputes everything; large repos take minutes on every run.

## Goal
Cache analysis results so an unchanged file is not re-analyzed. Done when a no-change re-run is under 5 seconds on a 1,000-file repo.

## Locked decisions
- ADR-0004: all persistence goes through the existing SQLite store; do NOT introduce a second datastore (the dual-store drift this team already paid for).

## Components
| # | Component | What |
|---|---|---|
| 001 | Cache | Store and look up per-file results |
| 002 | Invalidation | Detect changed files |

## Acceptance Criteria
- An unchanged file's cached result is reused.
- A changed file's cache entry is invalidated and recomputed.
- A no-change re-run reads zero results from source analysis.
