# epic.md

**Status:** Approved

## Problem
The public API has no rate limiting, so a single client can exhaust the request budget for everyone.

## Goal
A per-client token-bucket rate limiter. Done when a client over its limit gets HTTP 429 and a Retry-After header.

## Components
| # | Component | What |
|---|---|---|
| 001 | Bucket | Track tokens per client |
| 002 | Middleware | Enforce the limit on each request |

## Acceptance Criteria
- A client within its limit is allowed through.
- A client over its limit receives HTTP 429.
- A 429 response includes a Retry-After header in seconds.
