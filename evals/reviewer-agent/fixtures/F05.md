# epic.md

**Status:** Approved

## Problem
Importing a large project blocks the UI thread with no progress feedback.

## Goal
Show import progress in a notification while parsing runs off the UI thread.

## Architecture
Layered: `parser/` (pure, no platform APIs) is called by `commands/importer.ts`, which is called by `extension.ts`.
**Boundary rule:** only `src/extension.ts` may import the editor API (`vscode`); every other module receives what it needs via injected interfaces.

## Components
| # | Component | What |
|---|---|---|
| 001 | Parser | Parse the project files (pure) |
| 002 | Progress | Surface progress to the user |

## Acceptance Criteria
- Parsing a 500-file project reports progress at least every 100 files.
- The UI thread stays responsive during the import.
- Cancelling the import stops parsing within one file.
