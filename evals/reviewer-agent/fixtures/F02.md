# epic.md

**Status:** Approved

## Problem
Search over the notes list is slow and returns stale results after edits.

## Goal
An incremental search index that updates on note edits. Done when a query over 10,000 notes returns in under 50 ms and reflects the latest edit.

## Components
| # | Component | What |
|---|---|---|
| 001 | Index | Build and update the inverted index |
| 002 | Query | Resolve a query against the index |

## Acceptance Criteria
- A query returns every note whose body contains all query terms.
- Editing a note updates its index entry before the next query.
- The search should work well with large note collections.
